# FEM4C チュートリアル要件定義

本資料は、チュートリアルで「何をできるようにするか」を明確化するための要件定義です。

## 1. 目的
- 初学者が FEM を自力で実装できるようになること。
- 2D 線形静解析（単位厚さ）を、理論→実装→検算の流れで理解できること。
- FEM4C の構造を読んで自分の実装と対比できること。

## 2. 対象読者の前提
- C 言語の基礎（配列、ポインタ、構造体、ファイルI/O）。
- MATLAB または数値計算の基礎経験。
- FEM の用語（要素、節点、剛性、境界条件）を知っているが、実装経験は浅い。
- Chapter 02 の `practice/ch02/penalty.c` 程度の課題なら、時間はかかるが自走できる（必要に応じて外部に質問する前提）。

## 3. 学習到達目標
- T3 要素の形状関数、ヤコビアン、B/D 行列を説明できる。
- T3 の要素剛性を C で実装し、最小問題で検算できる。
- 全体剛性の組立、境界条件の適用、CG ソルバの流れを説明できる。
- FEM4C の主要モジュール（input/solver/elements/output）を追える。
- T6 要素の剛性・荷重を追加し、検算できる。
- 2 部品以上の入力を扱えるように拡張できる。

## 4. スコープ
- 対象解析: 2D 線形静解析（小変形）。
- 単位系: mm / N / N/mm^2 / kg/mm^3。
- 入力: native 形式、parser 出力パッケージ。
- 出力: 変位、応力、可視化（VTK/F06）。
- parser: NastranBalkFile の読解、T6/複数部品対応の拡張を含む。

## 5. スコープ外
- 非線形解析（材料・幾何）。
- 動解析・固有値解析。
- 接触、拘束、多体連成。
- 高度な最適化や並列分散。

## 6. 教材構成の要件
- 各章は「理論→実装→FEM4C確認→検証」の順で完結する。
- 1章1題の最小例に限定する。
- 例題の入力は最小構成を用い、結果の妥当性を必ず確認する。
- チュートリアル読了後に「T6 実装」「複数部品対応」の課題に接続する。

## 7. 検算基準
- 解析結果が既知解（理論解、単純問題）と一致すること。
- 変位が全てゼロになるケースは原因を特定して修正する。
- 単位系の整合性が取れていること。

## 8. コード配置の要件
- 演習・検証コードは `practice/` に配置する。
- `docs/` には手順、設計メモ、要件整理のみ置く。

## 9. 成果物
- 章ごとの最小実装コード（practice）。
- 章ごとの検算ログまたは確認手順。
- FEM4C の構造理解メモ（ドキュメント）。
- T6 実装の検算メモ。
- 複数部品対応の実装メモ。
- parser の拡張メモと検算ログ。

## 10. 学習ストーリーと時間目安
| フェーズ | ゴール | 目安時間 |
|---------|--------|----------|
| チュートリアル読了 + 例題 | T3 ソルバを自作し、FEM4C を追える | 35〜55 時間 |
| T6 実装 | T6 の剛性・荷重・検算を完了 | 14〜24 時間 |
| 複数部品対応 | 2 部品以上を解析できる | 14〜28 時間 |
| parser 対応 | parser の読解・拡張・検算 | 12〜20 時間 |
